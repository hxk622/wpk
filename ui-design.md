# UI设计规范

## 1. 设计原则

### 1.1 核心设计理念
- **简洁明了**：界面设计简洁，避免冗余元素，突出核心功能
- **直观易用**：操作流程符合用户习惯，降低学习成本
- **视觉层次**：通过色彩、排版和空间关系建立清晰的视觉层次
- **一致性**：保持设计风格和交互模式的一致性
- **响应式设计**：适配不同屏幕尺寸，提供良好的移动端体验

### 1.2 用户体验原则
- **高效交互**：减少用户操作步骤，提高交互效率
- **即时反馈**：对用户操作给予及时的视觉或听觉反馈
- **容错设计**：提供清晰的错误提示和恢复机制
- **个性化**：支持用户自定义界面主题和偏好

## 3. 排版

### 3.1 字体选择
- **主要字体**：Inter，无衬线字体，现代简洁
- **备用字体**：Helvetica，Arial，sans-serif
- **代码字体**：Monaco，Consolas，等宽字体

### 3.2 字体大小
| 元素类型 | 字体大小 | 行高 | 字重 |
|---------|----------|------|------|
| 大标题 | 28px | 36px | 700 |
| 小标题 | 24px | 32px | 600 |
| 三级标题 | 20px | 28px | 600 |
| 四级标题 | 18px | 26px | 500 |
| 正文文本 | 16px | 24px | 400 |
| 辅助文本 | 14px | 20px | 400 |
| 小字文本 | 12px | 16px | 400 |

### 3.3 字体样式
- **标题**：使用粗体，增强视觉层次感
- **正文**：使用常规字重，保证可读性
- **链接**：使用主色调，下划线可选
- **按钮文本**：使用粗体，确保清晰可见

## 4. 布局结构

### 4.1 网格系统
采用12列网格系统，适应不同屏幕尺寸：
- **移动端**：单列布局，间距16px
- **平板**：2-3列布局，间距20px
- **桌面端**：4-12列布局，间距24px

### 4.2 响应式断点
| 设备类型 | 屏幕宽度 | 布局策略 |
|---------|----------|----------|
| 移动端 | < 768px | 单列布局 |
| 平板 | 768px - 1024px | 双列或三列布局 |
| 桌面端 | > 1024px | 多列布局 |

### 4.3 间距规范
- **内边距**：卡片内边距16px，组件内边距8-16px
- **外边距**：组件外边距8-24px，区块间距24-48px
- **行高**：文本行高1.5-1.8倍，确保可读性

## 5. 组件设计

### 5.1 按钮组件
- **主要按钮**：使用主色调，圆角4px，高度40px
- **次要按钮**：使用边框样式，圆角4px，高度40px
- **文本按钮**：仅文本样式，无背景色
- **图标按钮**：仅图标，圆形或方形
- **按钮状态**：正常、悬停、点击、禁用

### 5.2 输入组件
- **文本输入框**：圆角4px，高度40px，边框1px
- **密码输入框**：带密码可见性切换功能
- **选择器**：下拉选择框，支持搜索和多选
- **日期选择器**：日历样式，支持日期范围选择
- **输入状态**：正常、聚焦、错误、禁用

### 5.3 卡片组件
- **基础卡片**：白色背景，阴影效果，圆角8px
- **带标题卡片**：包含标题区域和内容区域
- **可折叠卡片**：支持展开/折叠功能
- **带操作卡片**：包含操作按钮区域

### 5.4 导航组件
- **顶部导航栏**：固定在页面顶部，包含logo、菜单和用户信息
- **侧边导航**：固定在页面左侧，支持多级菜单
- **底部导航栏**：移动端使用，包含核心功能入口
- **面包屑导航**：显示当前页面位置，支持返回上级

### 5.5 弹窗组件
- **模态框**：居中显示，半透明背景遮罩
- **抽屉式弹窗**：从右侧或底部滑出
- **提示框**：顶部或底部显示，自动消失
- **确认对话框**：包含确认和取消按钮

### 5.6 游戏相关组件
- **扑克牌组件**：显示扑克牌，支持翻转效果
- **筹码组件**：显示不同面额的筹码
- **玩家信息组件**：显示玩家头像、昵称、筹码数量
- **操作按钮组件**：下注、跟注、加注、弃牌等游戏操作按钮
- **游戏状态组件**：显示当前游戏阶段、底池大小等信息

## 6. 页面设计

### 6.1 登录/注册页面
- **布局**：居中布局，简洁明了
- **元素**：Logo、标题、输入框、按钮、链接
- **交互**：表单验证、密码可见性切换、第三方登录

### 6.2 首页
- **布局**：顶部导航栏 + 轮播图 + 功能模块 + 底部信息
- **功能模块**：热门房间、推荐游戏、排行榜、活动公告
- **交互**：轮播图自动播放、卡片hover效果、按钮点击反馈

### 6.3 房间列表页面
- **布局**：顶部筛选栏 + 房间列表 + 底部操作栏
- **房间卡片**：显示房间名称、人数、底注、盲注等信息
- **交互**：筛选、排序、创建房间、加入房间

### 6.4 游戏房间页面
- **布局**：游戏区域 + 玩家区域 + 操作区域 + 聊天区域
- **游戏区域**：显示公共牌、底池、当前玩家等信息
- **玩家区域**：显示所有玩家的头像、昵称、筹码、手牌等信息
- **操作区域**：显示游戏操作按钮和下注滑块
- **聊天区域**：实时聊天功能，支持表情和快捷短语

### 6.5 个人中心页面
- **布局**：侧边导航 + 内容区域
- **功能模块**：个人信息、游戏记录、好友列表、设置
- **交互**：信息编辑、记录查询、好友管理、设置修改

### 6.6 AI分析页面
- **布局**：数据可视化图表 + 分析报告 + 历史记录
- **数据分析**：牌力分析、对手行为分析、胜率统计
- **交互**：图表切换、数据筛选、报告导出

## 7. 交互设计

### 7.1 基本交互模式
- **点击**：按钮、链接、卡片等可点击元素
- **滑动**：轮播图、滑块、列表滚动
- **拖拽**：筹码拖动、卡片排序
- **长按**：菜单弹出、操作确认

### 7.2 微交互设计
- **按钮悬停效果**：颜色变化、轻微放大
- **输入框聚焦效果**：边框颜色变化、阴影效果
- **页面切换动画**：淡入淡出、滑动过渡
- **加载状态**：骨架屏、加载动画
- **成功/失败提示**：顶部或底部消息提示

### 7.3 游戏交互设计
- **牌局流程动画**：洗牌、发牌、翻牌等动画效果
- **下注操作反馈**：筹码移动动画、金额变化
- **胜负判定动画**：胜利/失败提示，筹码分配动画
- **AI分析结果展示**：实时数据更新，图表动画

## 8. 视觉设计规范

### 8.1 图标设计
- **图标风格**：线性图标，统一的线条粗细（2px）
- **图标尺寸**：16px、24px、32px、48px
- **图标颜色**：主色调或中性色，根据上下文调整

### 8.2 图片设计
- **图片格式**：优先使用WebP格式，其次PNG和JPEG
- **图片质量**：压缩后保证视觉清晰，文件大小适中
- **图片尺寸**：根据实际需求设置，避免过大或过小
- **图片alt文本**：所有图片必须添加alt文本，提高可访问性

### 8.3 动画设计
- **动画时长**：200-500ms，避免过长影响体验
- **动画曲线**：使用ease-in-out缓动函数，使动画更自然
- **动画效果**：淡入淡出、滑动、缩放、旋转等
- **动画触发条件**：页面加载、用户操作、状态变化

### 8.4 无障碍设计
- **颜色对比度**：确保文本与背景的对比度符合WCAG标准
- **键盘导航**：支持纯键盘操作，tab键顺序合理
- **屏幕阅读器支持**：添加适当的ARIA属性，确保屏幕阅读器可识别
- **字体大小可调**：支持用户调整字体大小
