## 计划：启动前后端服务并在浏览器上测试

### 1. 启动后端服务

* **命令**：`cd packages/backend && npm run dev`

* **端口**：3000

* **验证**：访问 <http://localhost:3000/ping> 应该返回成功响应

### 2. 启动前端服务

* **命令**：`cd packages/frontend && npm run dev`

* **端口**：5173

* **验证**：服务启动后会自动打开浏览器，或手动访问 <http://localhost:5173>

### 3. 浏览器测试流程

#### 3.1 访问前端应用

* 打开浏览器访问：<http://localhost:5173>

* 预期结果：看到应用首页，显示登录/注册选项

#### 3.2 注册新用户

* 点击"注册"按钮

* 填写注册信息并提交

* 预期结果：注册成功，跳转到登录页面

#### 3.3 登录应用

* 使用注册的账号登录

* 预期结果：登录成功，跳转到房间列表页面

#### 3.4 创建房间

* 点击"创建房间"按钮

* 设置房间参数（包括测试最大玩家数量选择器的鼠标滚轮功能）

* 点击"创建房间"按钮

* 预期结果：房间创建成功，跳转到游戏房间页面

#### 3.5 测试房间列表

* 返回房间列表页面

* 预期结果：能看到刚才创建的房间，显示正确的房间信息

#### 3.6 测试最大玩家数量选择器

* 进入创建房间页面

* 点击最大玩家数量选择框

* 使用鼠标滚轮上下滚动

* 预期结果：能够通过鼠标滚轮选择不同的玩家数量

### 4. 验证前后端通信

* 检查浏览器控制台，确认没有网络错误

* 查看后端日志，确认API请求被正确处理

* 预期结果：前后端通信正常，没有404或500错误

### 5. 清理资源

* 测试完成后，停止前后端服务

* 命令：按 Ctrl+C 停止运行中的服务
